# SMS Автозаповнення - Посібник

## Як працює автозаповнення коду з SMS

### Для Android

1. **Дозволи**: Додаток опціонально запитує дозволи на читання SMS (`READ_SMS`, `RECEIVE_SMS`)
   - Дозволи не є обов'язковими для роботи додатку
   - Користувач може відхилити дозволи і вводити код вручну
2. **Формат SMS**: SMS повинно містити 6-значний код у форматі:
   ```
   Ваш код підтвердження для Zeno: 123456
   ```
3. **Автозаповнення**: Android автоматично розпізнає код і запропонує його для автозаповнення

### Для iOS

1. **Формат SMS**: SMS повинно містити код у форматі:
   ```
   Ваш код підтвердження для Zeno: 123456
   ```
2. **Автозаповнення**: iOS автоматично розпізнає код і запропонує його для автозаповнення

### Безпека та приватність

- **Опціональні дозволи**: Додаток працює навіть без дозволів на SMS
- **Пояснення**: Користувачу пояснюється навіщо потрібні дозволи
- **Вибір**: Користувач може відхилити дозволи і використовувати ручне введення
- **Безпека**: SMS дані не зберігаються та не передаються третім особам

## Технічні деталі

### Віджет SixDigitPinInput

- Використовує `AutofillHints.oneTimeCode` для автоматичного розпізнавання
- Підтримує автозаповнення з клавіатури
- Автоматично викликає `onPinCompleted` при введенні 6 цифр

### Сторінки з автозаповненням

1. **OtpPage** - основна сторінка верифікації OTP
2. **PinRecoveryOtpPage** - сторінка відновлення PIN-коду

### Налаштування SMS

Для правильної роботи автозаповнення, SMS повинно містити:

1. **Текст з кодом**: "Ваш код підтвердження для Zeno: 123456"
2. **6-значний код**: Код повинен бути саме 6 цифр
3. **Читабельний формат**: Код повинен бути легко розпізнаваний системою

## Примітки

- Автозаповнення працює тільки на реальних пристроях
- На емуляторах може не працювати через відсутність SMS
- **Дозволи SMS є опціональними** - додаток працює без них
- iOS автоматично підтримує автозаповнення без додаткових налаштувань
- **Безпека**: SMS дані не зберігаються та не передаються третім особам

## Рішення проблеми блокування додатку

### Проблема
> "Приложение заблокировано для защиты устройства, запрашивает доступ к данным деликатного характера"

**Причина**: При встановленні APK файлу напряму (не через Google Play), Android блокує додатки з дозволами SMS.

### Рішення
1. **Повне видалення дозволів SMS**: Дозволи SMS повністю видалені з AndroidManifest.xml
2. **Використання системних можливостей**: Flutter використовує вбудовані можливості Android/iOS для автозаповнення
3. **AutofillHints**: Використовується `AutofillHints.oneTimeCode` для автоматичного розпізнавання коду
4. **Безпека**: Додаток не має доступу до SMS, але все ще підтримує автозаповнення

### Результат
- ✅ **Додаток встановлюється без блокування** - немає дозволів SMS
- ✅ **Автозаповнення все ще працює** - через системні можливості
- ✅ **Безпека збережена** - додаток не має доступу до SMS
- ✅ **Працює на всіх пристроях** - як при встановленні APK, так і через Google Play

### Як це працює без дозволів SMS:
- **Android**: Використовує системні можливості для розпізнавання OTP коду
- **iOS**: Автоматично розпізнає код з SMS через вбудовані механізми
- **Flutter**: `AutofillHints.oneTimeCode` активує системне автозаповнення
